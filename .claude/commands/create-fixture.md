---
description: Create a new E2E test fixture skill
---

Help me create a new test fixture skill in `e2e/fixtures/`. Ask me:
1. What should this skill do (its purpose)?
2. What vulnerabilities or qualities should it demonstrate?
3. Should it pass or fail the audit?

Then create a realistic skill directory with:
- `skill.json` manifest with appropriate metadata and permissions
- `src/index.ts` main implementation
- Any additional source files needed
- README.md and LICENSE if it's a "good" skill

The skill should look realistic, not like obvious test code. Vulnerabilities should be the kind real skills actually have.
